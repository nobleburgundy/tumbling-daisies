# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Static band website for Tumbling Daisies, deployed via GitHub Pages. No build step — HTML/CSS/JS files are served directly. Node scripts handle data fetching and are run locally or via CI.

## Scripts

```bash
npm run fetch-gigs       # Pull gigs from Google Calendar → gigs.json, bit-status.json
npm run update-music     # Scan assets/music/*.mp3 → music.json (reads ID3 tags)
./update-press-photos.sh # Resize press photos to 1200px thumbs, rebuild press-photos.json
```

`fetch-gigs` requires `.env` with:
- `GOOGLE_SERVICE_ACCOUNT_KEY` — JSON of a GCP service account with Calendar read access
- `BANDSINTOWN_API_KEY` — optional; skipped if absent

## Architecture

**Data flow:** Google Calendar → `scripts/fetch-gigs.js` → `gigs.json` → `js/main.js` → DOM

**Automation:** GitHub Actions (`.github/workflows/fetch-gigs.yml`) runs `fetch-gigs` every 12 hours and commits `gigs.json` + `bit-status.json` if changed.

**Frontend:** Single-page app feel via `js/main.js` — intercepts internal link clicks, fetches the target `.html` file, swaps `#page-content` innerHTML, updates history. The persistent audio player survives page swaps because it's created once in `main.js` and appended to `<body>` (not inside `#page-content`).

**JSON data files** (committed, generated by scripts):
- `gigs.json` — upcoming confirmed gigs parsed from Google Calendar events
- `music.json` — track list with filenames and album metadata from MP3 tags
- `press-photos.json` — filenames of press photos in `assets/`
- `hidden-gigs.json` — array of gig IDs to suppress from the public shows list
- `bit-status.json` — Bandsintown published event status
- `config.json` — calendar ID, band keywords, hold/confirmed status keywords, date window

**Gig metadata:** Extra fields (showtime, ticket links, lineup, cover charge, etc.) are stored as `Key: Value` lines in the Google Calendar event description. `fetch-gigs.js` parses these into structured fields and strips them from the description. `Pay:` lines are parsed but never written to `gigs.json`.

**Admin page** (`admin.html`): Local-only tool for reviewing gig metadata tags and managing `hidden-gigs.json`. Not linked from the public nav.

**Press photos:** Source images live in `assets/press-*.{jpg,jpeg,png}`. Run `./update-press-photos.sh` (macOS only — uses `sips`) after adding/removing photos to regenerate thumbs and `press-photos.json`.
